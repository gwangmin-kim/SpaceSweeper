# Development Roadmap: Space Sweeper

|**기간**|**단계**|**주요 목표**|
|---|---|---|
|**Week 1-2**|**Phase 1. 프로토타입**|이동, 채굴, 산소 제한 등 핵심 메카닉 검증|
|**Week 3-4**|**Phase 2. 코어 루프 & 데이터**|우주선(Hub) <-> 탐사 순환 구조 및 저장 시스템 구축|
|**Week 5-6**|**Phase 3. 콘텐츠 & 성장**|장비 업그레이드 로직, 맵 다양화, 위협 요소 추가|
|**Week 7-8**|**Phase 4. 폴리싱 & 배포**|사운드, 이펙트, UI 연출 강화 및 버그 수정|

---

## Phase 1: 프로토타입

기간: 1주차 ~ 2주차
목표: "우주에서 움직이고, 부수고, 죽는 것이 재미있는가?" 검증

### 1.1. 프로젝트 기초 공사

- [x] Unity 프로젝트 생성 및 Git 연동
- [x] 폴더 구조 세팅 (`Scripts`, `Prefabs`, `Sprites`, `Resources`)
- [ ] 기본 에셋(임시 스프라이트) 구비

### 1.2. 플레이어 컨트롤러

- [x] **이동 로직 구현:** `Rigidbody2D` 기반 가속/감속 (무중력 관성)
- [x] **회전 로직:** 마우스 커서 방향 바라보기
- [x] **카메라 추적:** Cinemachine 활용, 플레이어 부드럽게 따라가기
- [x] **충돌 처리:** 벽(소행성) 충돌 시 물리 반응

### 1.3. 채굴 및 자원 시스템

- [x] **채굴 액션:** 마우스 클릭 시 투사체 발사 or 히트스캔 공격
- [x] **파괴 가능한 오브젝트:** 체력을 가진 폐기물 구현, 파괴 시 파편 생성
- [x] **자원 수집:** 플레이어에게 끌려오는(Magnet) 자원 아이템 구현 및 획득 처리

### 1.4. 산소 및 게임오버

- [x] **산소 타이머:** 시간 경과에 따른 감소 로직
- [x] **UI 표시:** 현재 산소량 슬라이더/텍스트 표시
- [ ] **사망/귀환:** 산소 0 도달 시 게임오버 처리, 특정 지점(출구) 도달 시 성공 처리

## Phase 2: 코어 루프 & 데이터

기간: 3주차 ~ 4주차
목표: 탐사를 마치고 돌아와서 저장을 하고 다시 나가는 '순환' 완성

### 2.1. 씬 관리 (Scene Flow)

- [ ] **타이틀 화면:** [새 게임], [이어하기], [종료] 버튼 기능
- [ ] **씬 전환 매니저:** 타이틀 -> 우주선(Hub) -> 탐사(Game) -> 우주선 이동 구현
- [ ] **로딩 화면:** 비동기 씬 로딩 (`SceneManager.LoadSceneAsync`)

### 2.2. 데이터 매니저 (Data Architecture)

- [ ] **데이터 모델링:** `PlayerData`, `Inventory`, `Settings` 클래스 설계
- [ ] **싱글톤 매니저:** `GameManager`를 통한 전역 상태 관리
- [ ] **저장 시스템:** `JsonUtility` 활용, 파일 입출력(Save/Load) 구현

### 2.3. 우주선(Hub) 기능

- [ ] **UI 상호작용:** 교환소, 지도, 출구 버튼 UI 배치
- [ ] **자원 정산:** 탐사에서 가져온 자원을 Hub 인벤토리에 합산
- [ ] **상점 기초:** 자원을 소모하여 '재화'로 바꾸는 기능

## Phase 3: 콘텐츠 & 성장

기간: 5주차 ~ 6주차
목표: 플레이어가 강해지는 것을 체감하고, 새로운 도전을 제공

### 3.1. 업그레이드 시스템

- [ ] **업그레이드 UI:** 트리 형태의 버튼 배치 및 상태 표시 (잠김/해금/완료)
- [ ] **스탯 적용:** 이동 속도, 산소통 용량, 채굴 데미지 변수 연결
- [ ] **장비 교체:** 곡괭이 -> 샷건 -> 레이저건 무기 변경 로직

### 3.2. 맵 및 환경 확장

- [ ] **레벨 디자인:** 난이도별(가까운 궤도/먼 궤도) 맵 프리팹 제작
- [ ] **장애물 추가:** 움직이는 우주 쓰레기, 고정형 포탑 등 위협 요소
- [ ] **배경 연출:** Parallax(시차) 배경 스크롤링 적용

### 3.3. 사운드 시스템

- [ ] **SoundManager:** BGM, SFX(효과음) 분리 및 볼륨 조절 기능
- [ ] **리소스 적용:** 채굴음, 엔진음, 배경 음악 적용

---

## Phase 4: 폴리싱 & 배포

기간: 7주차 ~ 8주차
목표: "팔아도 될 정도"로 마감 품질 높이기

### 4.1. 비주얼 폴리싱
- [ ] **파티클 이펙트:** 추진기 불꽃, 채굴 시 파편 튀김, 폭발 효과
- [ ] **화면 효과:** 피격 시 화면 붉어짐, 산소 부족 시 경고 효과 (Post-Processing)
- [ ] **UI 애니메이션:** 버튼 클릭 피드백, 팝업 등장 연출

### 4.2. 설정 및 편의성

- [ ] **옵션 메뉴:** 해상도 조절, 볼륨 조절 기능 연결
- [ ] **키 바인딩:** Input System 활용하여 키 변경 기능 (선택 사항)

### 4.3. 최적화 및 빌드

- [ ] **프로파일링:** 프레임 드랍 구간 확인 및 최적화
- [ ] **최종 빌드:** Windows/Mac 실행 파일 생성 및 테스트
