### 🚀 트러블 슈팅
**문제 상황:**
점프 구현 시 `groundCheckOffset`을 중심으로 원형 영역을 검사하여 착지 여부(`_isGrounded`)를 판단하는데, 착지 시점에 수직 속도를 0으로 초기화하니 매 착지마다 높이가 미묘하게 달라지는 현상 확인.

**시도한 방법:**
1. 착지 시점에 지면과의 거리를 확인(`RayCast` 이용), 높이 차만큼 `MovePosition` 사용하여 바닥에 스냅
	- 바닥에 달라붙기는 하지만, 착지 시점에 좌우 이동이 미묘하게 끊기는 듯한 조작감이 좋지 않았음.
2. 지면에 닿아있을 때 수직 속도를 항상 -0.1로 설정(DownForce). 이 경우 바닥에 붙어있는 것은 `Rigidbody2D`의 충돌 매커니즘에 맡김으로써 항상 바닥에 붙어있을 수 있음.

**해결:**
2번 방법으로 해결.
- 이 경우 점프 구현에 또다른 이슈 발생: 지면에 닿아있는 동안 항상 음의 수직속도를 적용하니, 점프 직후 원형 영역에 아직 바닥이 남아있을 경우 곧바로 수직속도가 초기화되어 점프가 불가능해짐. 이에 두 가지 해결책 적용.
	1. 점프 직후 일정 시간(현재 0.1초로 설정) 동안 착지 여부를 검사하지 않음. 
	2. 수직 속도가 양수일 경우 착지 여부를 검사하지 않음.
- 2가 실질적인 해결책으로 보임. 다만, 1의 매커니즘도 유효하다고 판단하여 여전히 코드에는 유지되어 있음.
